1.准备工作

先安装好相关的工具，这里提供了brewhome安装和已经安装了Xcode的两种方式，下面两个二选一，这里墙裂推荐brewhome。

1.1方式一:安装第三方包管理brewhome

首先安装好第三方包管理工具brewhome，方便后续软件的管理和安装。
直接一句命令搞定

ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

1.2方式二:安装Xcode的命令行工具

提前安装好Xcode的 command line tools,同样一句话搞定

xcode-select --install

2.安装Git

安装git的方式比较多，这里只提供两种，二选一即可，如果已经安装可以跳过。账号注册比较简单，就直接略过。

2.1方式一:直接brewhome安装

依旧一句命令搞定

sudo brew install git

2.2方式二:直接命令行安装

前提已经安装好Xcode的Command Line Tools

curl -O http://kernel.org/pub/software/scm/git/git-1.7.5.tar.bz2
tar xjvf git-1.7.4.1.tar.bz2
cd git-1.7.4.1
./configure --prefix=/usr/local
make
sudo make install
which git

3.Github的ssh设置

3.1生成ssh的key

输入以下命令生成ssh的key，引号内是你的github邮箱，如果已经配置完成，同样跳过。

ssh-keygen -t rsa -C "xxx@xxx.com”

如果出现以下提示，直接回车就好

Generating public/private rsa key pair.
Enter file in which to save the key (/        your_home_path/.ssh/id_rsa):

接着会显示一下信息，提示你输入两次密码，这个是在提交项目时输入的密码，不想设置的话可以直接回车跳过

Enter passphrase (empty for no passphrase): [Type a passphrase]
Enter same passphrase again: [Type passphrase again]

最后如果出现一下信息就表示设置成功

Your identification has been saved in /your_home_path/.ssh/id_rsa.
Your public key has been saved in /your_home_path/.ssh/id_rsa.pub.
The key fingerprint is: 01:0f:f4:3b:ca:85:d6:17:a1:7d:f0:68:9d:f0:a2:db your_email@example.com

3.2添加SSH key到自己的Github上

1,在完成上一步后，会生成一个.ssh的隐藏文件，这里直接输入以下命令打开后全选复制里面的内容

cd ~/.ssh
open -e id_rsa.pub

2,接着登陆github，按照以下步骤

1,点击右上角的头像图标 Settings—->SSH keys —-> add SSH keys

2,把刚刚复制的内容粘贴到key里，title随便填

3,点击add SSH，这就完成了

3,本地测试，直接复制以下命令到命令行回车，不用修改下面的邮箱

ssh -T git@github.com

如果看到下面的提示，则是提醒你网站证书未验证而已，直接输入yes后回车就行

The authenticity of host 'github.com (207.97.227.239)' can't be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

最后看到以下提示，标明ssh的key设置成功

Hi username! You've successfully authenticated, but GitHub does not provide shell access.

最后设置一下自己的个人信息，用于push提交内容时记录提交者信息，就完成了，引号内的内容自行修改

git config --global user.name "your user_name"
git config --global user.email "your_email@example.com"

4.安装Node.js

同样提供两种方式，二选一即可

4.1方式一:直接brewhome安装

一句命令搞定

brew install node

4.2方式二:直接命令行安装

这里要求已经安装Xcode命令行工具和git,即已经完成以上步骤

git clone git://github.com/joyent/node.git
cd node
./configure
make
sudo make install

5.安装hexo

这里也提供两种方式，直接用node的npm安装和源码下载编译，以下同样二选一

5.1方式一:直接npm安装

完成以上步骤安装了node的情况下，同样一句话搞定

npm install -g hexo

5.2方式二:源码下载编译

直接使用以下命令下载编译源码

./configure --prefix=~/nodejs && make && make install
cp ~/nodejs/bin/node /usr/sbin/node
~/nodejs/bin/npm install -g hexo

6.生成hexo博客文件

完成以上步骤后，cd命令进入到要存放博客的目录,用以下命令生成hexo博客相关文件

hexo init

接着安装下依赖

npm install
npm install hexo-renderer-ejs --save
npm install hexo-renderer-stylus --save
npm install hexo-renderer-marked --save

接着修改hexo根目录下_config.yml文件（xxx为你的github账户名称）

deploy:
       type: git
       repo: git@github.com:xxx/xxx.github.io.git
      branch: master
      message: 博客迁移测试        #提交的注释内容

然后直接输入以下命令，在本地预览博文

hexo g
hexo s

如果查看没问题，直接输入以下命令提交

hexo d -g

注：因为目前用npm下载的hexo是最新的3.x版本，所以在配置完yml文件，执行 hexo deploy后,可能出现 error deployer not found:github 的错误，这时候直接 cd进入生成hexo的博文目录，输入以下命令即可

npm install hexo-deployer-git --save
